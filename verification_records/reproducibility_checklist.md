# Reproducibility and Audit Checklist

For any future extension of these notes, each case should include:

- [ ] exact problem statement;
- [ ] known baseline or previous result;
- [ ] prompt pattern used to generate candidates;
- [ ] candidate object or claim;
- [ ] verification method;
- [ ] status label;
- [ ] reason for rejection if rejected;
- [ ] whether professor/expert feedback was used;
- [ ] whether computation depends on code;
- [ ] whether code depends on finite-field or graph-library assumptions;
- [ ] whether citation claims were checked manually.

This checklist is deliberately stricter than ordinary note-taking because the study concerns AI-assisted research. The central risk is that plausible text can hide a false mathematical step.
